API Reference and Usage Guide for Justify.AI

API Usage

The Justify.AI API provides programmatic access to hate speech analysis capabilities. All API requests should be made to the base URL with appropriate authentication headers.

Base URL

https://api.justify-ai.com/v1

Authentication

All API requests require an API key passed in the Authorization header: 

Authorization: Bearer YOUR_API_KEY

Analyze Text Endpoint

Analyze text for hate speech patterns, causal relationships, and bias dimensions.

POST /analyze

Request Example

fetch('https://api.justify-ai.com/v1/analyze', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': 'Bearer YOUR_API_KEY'
  },
  body: JSON.stringify({
    text: "Your text to analyze here",
    language: "en",
    options: {
      includeLexicon: true,
      includeCausal: true,
      includeBias: true
    }
  })
})
.then(response => response.json())
.then(data => console.log(data));

Response Example

{
  "status": "success",
  "data": {
    "riskScore": 78,
    "riskLevel": "high",
    "hateWords": [...],
    "causalGraph": {...},
    "biasMetrics": {...}
  },
  "metadata": {
    "language": "en",
    "processingTime": 1.23,
    "timestamp": "2026-03-13T10:30:00Z"
  }
}

Data Format

Understanding the data structures returned by the API is essential for integrating Justify.AI into your applications.

Analysis Result Object

The main analysis result contains comprehensive information about detected hate speech. 

interface AnalysisResult {
  riskScore: number;           // 0-100 composite risk score
  riskLevel: string;           // "low" | "medium" | "high"
  hateWords: HateWord[];       // Array of detected hate words
  causalGraph: CausalGraph;    // Causal relationship structure
  biasMetrics: BiasMetrics;    // Multi-dimensional bias scores
  breakdown: {
    lexicalSeverity: number;   // 0-100
    causalNarrative: number;   // 0-100
    biasIndicator: number;     // 0-100
  };
}

Hate Word Object

Each detected hate word includes severity and contextual information. 

interface HateWord {
  word: string;                // The detected word
  severity: string;            // "low" | "medium" | "high" | "extreme"
  severityScore: number;       // 0-100 numerical severity
  context: string;             // Surrounding text context
  position: {
    start: number;             // Character start position
    end: number;               // Character end position
  };
  category: string;            // Hate category (e.g., "religious", "ethnic")
}

Causal Graph Object

The causal graph represents cause-and-effect relationships in the text. 

interface CausalGraph {
  nodes: CausalNode[];         // Array of causal nodes
  edges: CausalEdge[];         // Array of causal relationships
}

interface CausalNode {
  id: string;                  // Unique node identifier
  text: string;                // Node text content
  type: string;                // "cause" | "effect" | "entity"
  importance: number;          // 0-1 importance score
}

interface CausalEdge {
  source: string;              // Source node ID
  target: string;              // Target node ID
  relationship: string;        // Relationship type
  confidence: number;          // 0-1 confidence score
}

Bias Metrics Object

Bias metrics quantify bias across multiple dimensions. 

interface BiasMetrics {
  religion: number;            // 0-100 religious bias score
  gender: number;              // 0-100 gender bias score
  ethnicity: number;           // 0-100 ethnic bias score
  overall: number;             // 0-100 composite bias score
  details: {
    [dimension: string]: {
      score: number;
      terms: string[];         // Contributing terms
      confidence: number;      // 0-1 confidence
    };
  };
}

Lexicon Builder

The Lexicon Builder component automatically discovers and maintains hate speech lexicons across multiple languages using word embeddings and semantic similarity.

How It Works

  1. Seed Terms: Start with a curated set of known hate speech terms
  2. Embedding Space: Map terms to high-dimensional vector space using language-specific embeddings
  3. Similarity Expansion: Find semantically similar terms using cosine similarity
  4. Context Validation: Validate candidates using contextual usage patterns
  5. Severity Scoring: Assign severity scores based on usage frequency and context

Using the Lexicon Builder

You can use the lexicon builder programmatically to expand your own hate speech lexicons. 

from src.lexicon import LexiconBuilder

# Initialize builder with language
builder = LexiconBuilder(language='en')

# Load seed terms
seed_terms = ['hate', 'violence', 'discriminate']
builder.load_seeds(seed_terms)

# Expand lexicon
expanded_lexicon = builder.expand(
    similarity_threshold=0.7,
    max_terms=1000,
    validate_context=True
)

# Get terms with severity scores
for term in expanded_lexicon:
    print(f"{term.word}: {term.severity_score}")

Configuration Options

# Configure lexicon builder
config = {
    'embedding_model': 'fasttext',  # or 'word2vec', 'bert'
    'similarity_threshold': 0.7,     # 0-1, higher = stricter
    'context_window': 5,             # words before/after
    'min_frequency': 10,             # minimum corpus frequency
    'severity_method': 'contextual'  # or 'frequency', 'hybrid'
}

builder = LexiconBuilder(language='hi', config=config)

Causal Extractor

The Causal Extractor identifies cause-and-effect relationships within hate narratives using dependency parsing and semantic role labeling.

Extraction Process

  1. Dependency Parsing: Parse text to identify grammatical relationships
  2. Causal Pattern Matching: Identify causal connectives and patterns
  3. Entity Recognition: Extract entities involved in causal relationships
  4. Relationship Extraction: Build cause-effect pairs with confidence scores
  5. Graph Construction: Create directed graph of causal relationships

Using the Causal Extractor

from src.causality import CausalExtractor

# Initialize extractor
extractor = CausalExtractor(language='en')

# Extract causal relationships
text = "Religious hatred leads to violence and discrimination"
causal_graph = extractor.extract(text)

# Access nodes and edges
for node in causal_graph.nodes:
    print(f"Node: {node.text} (type: {node.type})")

for edge in causal_graph.edges:
    print(f"{edge.source} -> {edge.target} ({edge.relationship})")

Causal Patterns

The extractor recognizes various causal patterns including:

  • Explicit Connectives: "because", "therefore", "leads to", "causes"
  • Implicit Causality: Verb-based causation (e.g., "triggers", "results in")
  • Conditional Causality: "if...then" structures
  • Temporal Causality: Sequential events implying causation

Visualization

# Export graph for visualization
graph_data = causal_graph.to_dict()

# Use with vis.js or D3.js
import json
with open('causal_graph.json', 'w') as f:
    json.dump(graph_data, f, indent=2)

Bias Auditor

The Bias Auditor quantifies bias across multiple dimensions including religion, gender, and ethnicity using specialized bias lexicons and contextual embeddings.

Bias Dimensions

The system measures bias across three primary dimensions:

  • Religious Bias: Prejudice based on religious identity or beliefs
  • Gender Bias: Discrimination based on gender or sexual orientation
  • Ethnic Bias: Prejudice based on ethnicity, race, or national origin

Using the Bias Auditor

from src.bias import BiasAuditor

# Initialize auditor
auditor = BiasAuditor(language='en')

# Analyze text for bias
text = "Your text to analyze for bias"
bias_metrics = auditor.analyze(text)

# Access bias scores
print(f"Religion Bias: {bias_metrics.religion}")
print(f"Gender Bias: {bias_metrics.gender}")
print(f"Ethnicity Bias: {bias_metrics.ethnicity}")
print(f"Overall Bias: {bias_metrics.overall}")

# Get detailed breakdown
for dimension, details in bias_metrics.details.items():
    print(f"\n{dimension}:")
    print(f"  Score: {details['score']}")
    print(f"  Terms: {', '.join(details['terms'])}")
    print(f"  Confidence: {details['confidence']}")

Scoring Methodology

Bias scores are calculated using a multi-step process:

  1. Lexicon Matching: Identify bias-indicating terms from specialized lexicons
  2. Contextual Analysis: Analyze surrounding context using embeddings
  3. Intensity Scoring: Calculate intensity based on term frequency and context
  4. Dimension Aggregation: Combine scores across bias dimensions
  5. Normalization: Scale scores to 0-100 range

Custom Bias Lexicons

# Load custom bias lexicon
custom_lexicon = {
    'religion': ['term1', 'term2', ...],
    'gender': ['term1', 'term2', ...],
    'ethnicity': ['term1', 'term2', ...]
}

auditor = BiasAuditor(language='en')
auditor.load_custom_lexicon(custom_lexicon)

# Analyze with custom lexicon
bias_metrics = auditor.analyze(text)

Interpretation Guidelines

Score Range Level Interpretation
0-25 Low Minimal or no detectable bias
26-50 Moderate Some bias indicators present
51-75 High Significant bias detected
76-100 Extreme Severe bias with strong indicators

Rate Limits & Best Practices

Rate Limits

API requests are subject to the following rate limits:

  • Free Tier: 100 requests per day, 10 requests per minute
  • Basic Tier: 1,000 requests per day, 50 requests per minute
  • Pro Tier: 10,000 requests per day, 200 requests per minute
  • Enterprise: Custom limits based on agreement

Best Practices

  • Batch Processing: Combine multiple texts when possible to reduce API calls
  • Caching: Cache results for identical text inputs to avoid redundant analysis
  • Error Handling: Implement exponential backoff for rate limit errors
  • Language Detection: Pre-detect language to optimize processing
  • Text Preprocessing: Clean and normalize text before sending to API

Error Handling Example

async function analyzeWithRetry(text, maxRetries = 3) {
  for (let i = 0; i < maxRetries; i++) {
    try {
      const response = await fetch('https://api.justify-ai.com/v1/analyze', {
        method: 'POST',
        headers: {
          'Content-Type': 'application/json',
          'Authorization': 'Bearer YOUR_API_KEY'
        },
        body: JSON.stringify({ text, language: 'en' })
      });

      if (response.status === 429) {
        // Rate limit hit, wait and retry
        const retryAfter = response.headers.get('Retry-After') || Math.pow(2, i);
        await new Promise(resolve => setTimeout(resolve, retryAfter * 1000));
        continue;
      }

      if (!response.ok) {
        throw new Error(`API error: ${response.status}`);
      }

      return await response.json();
    } catch (error) {
      if (i === maxRetries - 1) throw error;
    }
  }
}
