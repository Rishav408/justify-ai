Schema & Validation Discipline

We enforce a 17-field schema with strict validation checks so that every row is consistent, explainable, and auditable. This schema drives downstream lexicon, bias, and risk modules.

text label severity is_implicit is_sarcasm target_group target_type directness call_to_action tone emotion profanity_count platform domain region confidence annotator_id

Allowed Labels

hate, non_hate, offensive, health_issue

Confidence Bounds

Strictly within 0.70 to 1.00 to keep annotation reliability high.

Boolean Integrity

is_implicit, is_sarcasm, call_to_action validated as true/false only.

Tone + Emotion

Tone in {neutral, aggressive, dismissive, sarcastic, fearful} with separate emotion tracking.

Training Chamber: Classical NLP Stack

We use classical NLP with interpretable Naive Bayes classifiers per language. The pipeline keeps the model explainable while scaling to five languages.

Preprocessing

Language-specific tokenization, stemming/lemmatization, stopword filtering.

Feature Extraction

BoW + N-grams with CountVectorizer (max_features = 5000).

Naive Bayes

MultinomialNB for explainable classification and fast iteration.

Evaluation

80/20 stratified split with precision, recall, F1, accuracy per language.

Scaling Across Five Models

We train five independent models and vectorizers, one per language, to preserve linguistic nuance and allow per-language calibration.

Explainable Pipeline Conveyor

Seven backend phases drive the analysis, from language detection to risk scoring and dashboard delivery.

Phase 1: Unified analysis schema + 5 language coverage.
Phase 2: Lexicon matching with severity + spans.
Phase 3: Causality extraction for cause ? effect narratives.
Phase 4: Bias auditing with demographic co-occurrence.
Phase 5: Risk scoring (0–100) with breakdowns.
Phase 6: Frontend wiring + live analysis output.
Phase 7: QA + validation tests.
